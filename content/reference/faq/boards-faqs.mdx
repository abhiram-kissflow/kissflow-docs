### Can I handle multiple use cases, such as issue tracking, help desk, and service requests, using Kissflow Board?

Yes. Kissflow Board is built to support a wide range of workflows on a single platform — project management, issue tracking, bug tracking, help desks, service requests, ticketing systems, and more.

### There are a few boards I use frequently. How can I access them quickly without going to the Explore page?

Pin them to the left navigation. On the board, click the icon next to the board's name to pin it. Pinned boards then appear in the left navigation for quick access.

### I'd like to add custom statuses to my board. Is that possible?

Yes. You can add any number of [custom statuses](/docs/build/boards/configuring-statuses) relevant to your workflow from **Settings**.

### What happens when I rename a custom status?

The new name is applied across the board immediately. Items currently assigned to that status show the new name — there's nothing to update on the items themselves.

### I no longer need one of the statuses in my workflow. Can I delete it?

Yes. Delete custom statuses you created from inside the **Edit workflow** menu. If items are still assigned to a status you delete, you're prompted to confirm deleting those items along with the status — there's no option to move them to another status first, so reassign any items you want to keep before deleting the status.

### Can I customize the board form? If so, how?

Yes. [Customize the form](/docs/build/boards/configuring-a-board-form) to capture information relevant to your workflow. Drag and drop fields, or add new sections and tables, using the form editor.

### I no longer want priority for the items in my board, since this field doesn't apply to my use case. What do I do?

[Turn off priority](/docs/build/boards/configuring-your-board-settings) in settings. You can do this even if priority was already used to create items or set up other features such as filters and views.

### Can I create custom filters to track certain items frequently?

Yes — use [Views](/docs/build/boards/creating-and-using-board-views) inside the board to create customized views of your data. Saved views appear in the **Views** dropdown, and you can also pin them as tabs for quick access.

### I see items can be retrieved using different options such as "Created by me" and "Requested by me." Aren't these the same?

No, they show different data. **Created by me** lists items you created in the board. **Requested by me** lists items you created on someone else's behalf — for example, if you use a board as a help desk, you often "create" requests that a customer "requested."

### I want to find out the time spent on each status for an item. How can I do that?

Open the item, click the **Item log** tab in the right navigation strip, and select the **Status log** subtab. It captures every status change for the item along with the time spent in each status. See [Collaborating on items](/docs/use/items/collaborating) for more.

### Where can I find a history of actions performed on an item?

Open the item, click the **Item log** tab in the right navigation strip, and select the **Activity log** subtab. It lists key actions performed on the item — such as changes to system fields, with the value before and after each change — plus when the item was created, closed, or cloned. See [Collaborating on items](/docs/use/items/collaborating) for the full list of what's tracked.

### How do I determine the resolution time of an item?

Resolution time is tracked as a system field for items in the **Resolved** or **Closed** status, and is available as a column in views and reports on the board.

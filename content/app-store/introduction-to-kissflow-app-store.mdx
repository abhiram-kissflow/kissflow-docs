# Introduction to Kissflow App Store

<PlanBadge plans={["enterprise"]} />

Kissflow App Store is a dedicated place that hosts a wide array of pre-built applications for common business use cases. Unlike the process and board templates in Kissflow's template gallery, App Store apps are built by Kissflow specialists and are ready to install with little to no coding effort — see [Introduction to Kissflow Apps](/docs/build/apps/introduction-to-kissflow-apps) for how the underlying app platform works. Once you install one, you can deploy it to your end users right away.

![Kissflow App Store home page listing available apps](/migration-assets/introduction-to-kissflow-app-store/1.png)
{/* TODO: verify screenshot matches current product UI */}

You can still [build your own app](/docs/build/apps/creating-an-app) on the Work Platform when you need something custom. But when an App Store app already matches your use case, it cuts down the time and effort further — installed apps come with role-based views and dashboards already set up, so there's little left to configure before people can start using them.

<Callout type="info">
  App Store apps come in two flavors: some install directly, and some are provisioned for you after you send in a request. See [Types of apps in Kissflow App Store](/docs/app-store/types-of-apps-in-kissflow-app-store) for the difference.
</Callout>

## Why use the App Store

Building an app from scratch makes sense when your use case is specific to how your team works. But for common, well-understood processes, an App Store app gets you to a working solution much faster:

- **Early-stage teams** get a working app for a common process without pulling focus away from growing the business.
- **Growing teams** get a reliable, ready-made app for the operations that need to stay stable through a growth phase.
- **Larger organizations** get a starting point that's already built for scale, which can shorten the path to automating a given business function.

In every case, the trade-off is the same: an App Store app gets you running quickly, while a custom build on the Work Platform gets you an app shaped exactly around your process.

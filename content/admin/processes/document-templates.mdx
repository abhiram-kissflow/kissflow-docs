# Document templates

<PlanBadge plans={["basic", "enterprise"]} />

A document template lets you automate the generation of process-related PDFs. For example, as an HR manager managing the **Recruitment** process, you can create a document template that generates offer letters automatically once an interview is closed.

<Callout type="info">
  You must be a [Process Admin](/docs/admin/processes/managing-process-members-and-permissions#process-admin) or a [Process Developer](/docs/admin/processes/managing-process-members-and-permissions#developer) to create a document template.
</Callout>

## Creating a document template

You can create a document template from scratch or generate a first draft with Kissflow AI.

### Create from scratch

1. Navigate to the process for which you want to create a document template.
2. Click **Manage** > **Document templates** > **Create template** > **Create from scratch**.
3. Enter a name and description for your document template.
4. Click **Create from scratch** to open the template in the editor and build it manually.

![](/migration-assets/document-templates/4.gif)
{/* TODO: verify screenshot matches current product UI */}

### Generate using AI

1. Navigate to the process for which you want to create a document template.
2. Click **Manage** > **Document templates** > **Create template** > **Generate using AI**.
3. Enter a document template name — for example, **Employee Offer Letter** or **Purchase Order**.
4. Select a language for the template.
5. Click **Generate**. Kissflow AI uses these inputs as a prompt to create the first draft, called **Draft 1**.

![](/migration-assets/document-templates/3.gif)
{/* TODO: verify screenshot matches current product UI */}

#### AI regeneration to enhance drafts

Refine a draft by adding more information to it, then click **Regenerate** to get the changes reflected in a new draft.

![](/migration-assets/document-templates/5.gif)
{/* TODO: verify screenshot matches current product UI */}

<Callout type="info">
  You can navigate to any existing draft and use it as the base for regenerating a new draft with the changes you want.
</Callout>

<Callout type="warn">
  A draft can only be used to regenerate within one hour of its creation. For example, if draft 7 was generated on January 19 at 3:45 PM, you can use it to regenerate a new draft only before January 19, 4:45 PM. You can also create a maximum of 10 drafts per document template.
</Callout>

#### Finalizing a template draft

Once you're satisfied with a draft, open it and click **Use this draft** at the top right of the page. The draft is transferred to the document template editor, where you can further customize the document.

## Editing a document template

The template editor supports the standard formatting options you'd expect from any text editor, plus support for [inserting process fields](#inserting-process-fields), [page breaks](#inserting-page-breaks), [images](#inserting-an-image), and [form tables](#adding-a-table).

![](/migration-assets/document-templates/1.png)
{/* TODO: verify screenshot matches current product UI */}

<Callout type="idea">
  This feature is built on the Jinja templating engine. If you're familiar with Jinja, you can write advanced conditions or loops to extend what your template can do — see the [Jinja documentation](https://jinja.palletsprojects.com/en/3.0.x/) for the full syntax.
</Callout>

### Inserting process fields

Insert a system or process field in either of these ways:

- Type two curly braces without a space between them (**&#123;&#123;**) to see the list of fields used in the current process, then select the one you want to insert.
- Drag and drop a field from the left panel onto the template editor.

![](/migration-assets/document-templates/19.gif)
{/* TODO: verify screenshot matches current product UI */}

**Supported fields**

Only form tables and the following field types are supported inside document templates:

- Text
- Text area
- Number
- Currency
- Email
- Sequence number
- Date
- DateTime
- User
- Dropdown
- Yes/No
- Radio button
- Image
- Aggregation
- Signature
- Scanner

**Unsupported fields**

These field types are not supported inside document templates:

- Multi-select dropdown
- Checkbox
- Lookup
- Remote lookup
- Attachment
- Smart attachment
- Geolocation
- Checklist
- Rating
- Slider
- Rich text
- Button

### Inserting page breaks

Add as many pages as you want in your document. Click the **Page break** button (![](/migration-assets/document-templates/7.svg)) in the toolbar to insert one.

![](/migration-assets/document-templates/2.png)
{/* TODO: verify screenshot matches current product UI */}

### Adding a table

Insert a form table created as part of your process into your document template the same way you'd insert a field — by dragging and dropping it onto the template editor.

<Callout type="warn">
  Dragging a form table onto the editor adds a line of code above and below it. These lines are added by Kissflow — don't remove or modify them, or the template will stop working correctly.
</Callout>

Enter the column header names, and use double curly braces to insert a process field as a column's value.

![](/migration-assets/document-templates/21.gif)
{/* TODO: verify screenshot matches current product UI */}

<Callout type="info">
  You can't drag and drop a field while setting a value for a specific field in a table — use double curly braces instead.
</Callout>

### Inserting an image

Add an image in any of these ways:

1. **From your desktop** — drag and drop an image file from your computer into the template editor.
2. **From a web page** — right-click the image and select **Copy image address**. In the template editor, click the **Insert image** icon, right-click and paste the image address, then click **Done**. Hover over the image to format, resize, or delete it.
3. **Copy and paste** — copy an image from an existing document (for example, in Microsoft Word or Google Docs) and paste it directly into the template editor.

![](/migration-assets/document-templates/20.gif)
{/* TODO: verify screenshot matches current product UI */}

<Callout type="idea">
  You can also copy and paste text content from any external source into the editor.
</Callout>

### Inserting a header and footer

A header appears at the top of a PDF document and a footer at the bottom. Both are typically used for information that repeats on every page, such as the document name, headings, page numbers, images, or the date.

1. Click **Header & Footer** in the editor to open a separate editor window for the header and footer. The height of the header and footer is set by the top and bottom margins of the base canvas — change these using the **Margins** button. Inside the header/footer editor, you can further configure a top and bottom margin, up to a maximum of 400px. The base canvas also determines the header and footer's left and right margins, and these can't be changed from inside the editor.
2. Add content to the header and footer:
   - Textual content
   - Images and static tables
   - Page number and page count

   <Callout type="info">
     You can't insert a child table in the header or footer.
   </Callout>

3. Once you've added the content you need, click **Save**, then click **Preview** to see how the document template will render.

## Formatting currency, number, and date fields

If your template includes a **Number**, **Currency**, **Date**, or **DateTime** [field](/docs/build/forms/basic-and-advanced-fields), you can control how its values display using the **Format** option. Three formatting categories are available: **Currency**, **Number**, and **Timezone**.

![](/migration-assets/document-templates/12.gif)
{/* TODO: verify screenshot matches current product UI */}

<Callout type="warn">
  Format changes apply to every field of that type inside the document template — they can't be localized to one particular field.
</Callout>

### Currency

Choose whether currency values show the currency code after the value, or the currency symbol before it.

![](/migration-assets/document-templates/6.png)
{/* TODO: verify screenshot matches current product UI */}

### Number

Choose how numbers are separated inside **Number** and **Currency** fields. Four separator styles are available, with the American style applied by default.

![](/migration-assets/document-templates/10.png)
{/* TODO: verify screenshot matches current product UI */}

### Date and time zone

Choose the time zone and date order used to display **Date** and **DateTime** fields. A preview of the expected result appears above the formatting options as you choose.

![](/migration-assets/document-templates/8.png)
{/* TODO: verify screenshot matches current product UI */}

## Customizing your page

Use the **Page setup** option to configure the basic layout and appearance of your document.

### Page orientation

Choose **Portrait** or **Landscape** for your pages.

### Page color

Set the background color of your pages.

### Margins

Configure your page margins:

- **Auto** — automatically sets all four margins (top, bottom, left, right) to 92px.
- **Custom** — lets you set each margin manually.

Click **Apply** to save your changes.

![](/migration-assets/document-templates/16.gif)

## Associating a document template with a process connector

Automate PDF generation by connecting your process to a document template through the [process connector](/docs/build/integrations/connectors/kissflow-process-connector).

1. In the connector, select the **Generate a PDF document** step.
2. Set up a connection with the required account.
3. Choose a process, then select the relevant process instance ID.

   ![](/migration-assets/document-templates/13.png)
   {/* TODO: verify screenshot matches current product UI */}

4. Select the document template to map to the process.
5. Enter a title for the document, and choose whether to attach it to a field on the process form. If yes, select the attachment field to use.

   <Callout type="info">
     You can refresh fields to see any recently added form fields. You can also map fields from the trigger step or from earlier action steps — these values are filled in automatically once the action runs.
   </Callout>

6. Click **Next**.
7. Add a [Kissflow Email connector](/docs/build/integrations/connectors/email-connector) step to share the generated PDF as an email attachment.

   ![](/migration-assets/document-templates/17.png)
   {/* TODO: verify screenshot matches current product UI */}

8. Fill in the email fields, and under **Attachments** > **Fields** > **Generate PDF**, select the **key** field.

   ![](/migration-assets/document-templates/18.png)
   {/* TODO: verify screenshot matches current product UI */}

9. Toggle the integration **ON** to activate it.

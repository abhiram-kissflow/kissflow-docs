# Reviewing publish changes

<PlanBadge plans={["basic", "enterprise"]} />

The **Review changes** feature gives Process admins and Developers complete visibility into every update made to a process before it goes live. Instead of manually comparing versions or tracking edits across sessions, you get a clear, consolidated summary of what changed.

<iframe src="https://player.vimeo.com/video/1138389042?app_id=122963" title="Review changes in Kissflow Process" className="aspect-video w-full rounded-lg border" allowFullScreen />

## Open Review changes

Inside the process editor, you can review changes in two ways:

1. **While editing** — Click **Review changes** at the top-right corner at any time during your editing session, without needing to publish.
2. **Before publishing** — Click **Go live** at the bottom-right, then select **Review changes** in the confirmation pop-up.

![Go live confirmation pop-up with the option to review changes before publishing](/migration-assets/reviewing-publish-changes/2.gif)

## What the summary shows

The **Review changes** page lists everything that's been **added**, **deleted**, or **modified** in your form, workflow, and permissions since the last published version.

![Review changes page showing a consolidated summary of form, workflow, and permission changes](/migration-assets/reviewing-publish-changes/1.png)

<Callout type="info">
  On the Enterprise plan, complex changes are also explained in plain language using **AI-powered summaries**.
</Callout>

## Discard changes

If some updates are no longer needed, click **Discard changes** on the **Review changes** page. This removes all draft updates made after the last published version and restores the process to its previously published state.

## Reviewing changes made by another user

When you [take over editing](/docs/admin/processes/publishing-editing-duplicating-archiving-and-deleting-a-process#edit-a-process) from another Process admin or Developer, you're first asked to review the changes made since the last published version. Choose:

- **Discard changes and take over editing** — removes all unpublished changes made by the other user and lets you start fresh from the last published version.
- **Save changes and take over editing** — keeps the other user's changes as your draft so you can continue from where they left off. This doesn't publish the process — you'll still need to click **Go live** separately when you're ready.

![Take over editing pop-up with options to discard or save the other user's changes](/migration-assets/reviewing-publish-changes/3.gif)

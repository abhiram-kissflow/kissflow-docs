# Associated flows for processes

<PlanBadge plans={["basic", "enterprise"]} />

The **Associated flows** section shows how a process depends on, and is depended on by, other flows in your account — giving you a map of relationships to check before you rename fields, change permissions, or archive the process.

## View a process's associated flows

1. Open the process and click the **Manage** button at the top right.
2. Click **Associated flows**.

   <Callout type="info">
     **Associated flows** is available only to process admins and editors, and only while the process is **Live**.
   </Callout>

![Process Manage panel with the Associated flows tile](/migration-assets/associated-flows-for-processes/1.png)
{/* TODO: verify screenshot matches current product UI */}

There are two types of associations, shown as tabs on this page: **Looked up flows** and **Connected flows**.

## Looked up flows

[Lookup fields](/docs/build/forms/field-types/lookup-field) let a process pull and display data from other flows. The **Looked up flows** tab shows both directions of that relationship.

![Looked up flows tab showing the Flows looked up to and Flows looked up from lists](/migration-assets/associated-flows-for-processes/2.png)
{/* TODO: verify screenshot matches current product UI */}

### Flows looked up to

Shows which flows are looking up this process to fetch and display information.

**Example**: If another process references details like **Purchase Order IDs** or **Vendor Names** from the current process, that process appears under **Flows looked up to**.

Click **Configure** next to a flow in this list to open this process's [security settings](/docs/admin/processes/security-settings-for-flows), where lookup permissions are managed.

<Callout type="info">
  The **Configure** action is available only under **Flows looked up to**. To manage what the current process looks up from other flows, open the security settings of that other flow instead.
</Callout>

### Flows looked up from

Shows the flows that the current process is looking up.

**Example**: If your **Purchase Request** process pulls **Asset Types** or **Budget Codes** from a master dataset, that dataset appears under **Flows looked up from**.

## Connected flows

<Callout type="info">
  Connected flows require the Enterprise plan.
</Callout>

[Connected processes](/docs/admin/processes/connected-processes-overview) let actions in one process automatically create or update an item in another process. The **Connected flows** tab shows both directions of that relationship.

![Connected flows tab showing the Dependent flows and Source flows lists](/migration-assets/associated-flows-for-processes/3.png)
{/* TODO: verify screenshot matches current product UI */}

### Dependent flows

Shows the connected processes that can automatically initiate new items based on items created or updated in this process.

**Example**: When a **Purchase Request** item is approved, it can automatically create an item in a connected **Purchase Order** process.

### Source flows

Shows the connected processes that can initiate new items inside this process.

**Example**: A **Service Request** process can trigger this process to create a hardware request.

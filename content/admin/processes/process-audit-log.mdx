# Process audit log

<PlanBadge plans={["basic", "enterprise"]} />

The audit log stores the history of all activity related to a process, and lets [Process Admins](/docs/admin/processes/managing-process-members-and-permissions#process-admin) search flow-level and item-level changes made by users in the process.

<Callout type="info">
  The audit log is available only to Process Admins. If you don't see it, check with your Process Admin.
</Callout>

## Open the audit log

1. Open the process and click the **Manage** button.
2. Click **Audit log**.
3. Go to **Flow-level logs** to see changes to the process itself (settings, publishing, permissions), or **Item-level logs** to see changes to individual items within it.

![Process manage menu with the Audit log option](/migration-assets/process-audit-log/1.png)

The sorting, filtering, and export instructions below apply to both views. Each entry shows the following columns:

- **Audit ID** — a unique ID the system generates automatically for every recorded event.
- **Time** — when the action happened.
- **Acted by** — the user who performed the action.
- **[Event](/docs/admin/audit-log/event-categories-and-actions)** — the kind of activity that took place, for example **Process published**, **Item created**, or **Item deleted**.
- **Object** — the resource that was changed.
- **[Event category](/docs/admin/audit-log/event-categories-and-actions)** — a common name for a set of events, useful for filtering down to similar events. Examples include **Process Administration** and **Process Settings**.
- **Platform** — the device or OS the event was recorded from.
- **IP** — the IP address the event was recorded from.
- **Batch ID** — a unique ID the system generates automatically for a group of related events. Unlike **Audit ID**, every event in the same batch shares the same **Batch ID**.

## Sort the log

1. Click the down-arrow beside the **Time** column header.
2. Select **Ascending** or **Descending** to sort the log by time.

<Callout type="info">
  Sorting is available only on the **Time** column. The down-arrow on other columns lets you pin or unpin them instead, so key details stay visible while you scroll horizontally.
</Callout>

## Search a filter

Some filters — **Audit ID**, **Batch ID**, and **IP** — search by exact value:

1. Click the down-arrow beside the filter.
2. Enter the exact value in the search area that appears and apply it.

## Filter the log

You can filter audit log events on the following fields:

- **Audit period** — choose **Last 7 days**, **Last 30 days**, **Last 90 days**, or a **Custom date range**.

  <Callout type="info">
    **Last 30 days** may not be available on every account. If you don't see it, [contact Kissflow support](/docs/whats-new/important-notices/slas-and-support-models#how-to-reach-support) to check whether extended log retention can be enabled for your account.
  </Callout>

- **Object** — select one or more objects from the suggestions, or enter the exact object name.
- **[Event category](/docs/admin/audit-log/event-categories-and-actions)** — filtering by event category is one of the recommended ways to narrow down results. If you're not sure which category to pick, search for a flow name like "process" and the field lists every category relevant to it.

  ![Event category filter listing categories relevant to a flow](/migration-assets/process-audit-log/5.png)
  {/* TODO: verify screenshot matches current product UI */}

- **Acted by** — select one or more account users to filter events by who performed them.
- **Event** — enter or select one action from the suggested list. This filter accepts only one value at a time.
- **Audit ID, Batch ID, and IP** — enter the exact value to filter events, mainly useful to backtrace a specific event's history.

Each applied filter shows a control to clear it, and you can click **Clear all** to reset every filter at once.

![Audit log with filters applied and a clear control on each](/migration-assets/process-audit-log/6.png)
{/* TODO: verify screenshot matches current product UI */}

## Export the audit log

You can export up to 90 days of the audit log at a time.

1. Open the process and click **Manage** > **Audit log**.
2. Go to **Flow-level logs** or **Item-level logs**, depending on which log you want to export — see [Open the audit log](#open-the-audit-log) above for what each view covers.
3. Click the **Export** button and confirm the export.

<Callout type="info">
  Export runs as a background job. Once it finishes, you'll receive an email with the download link at your registered email address — this can take a few minutes rather than being instant.
</Callout>

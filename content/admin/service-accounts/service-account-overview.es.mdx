## ¿Qué es una cuenta de servicio?

Una cuenta de servicio es una cuenta que no pertenece a un usuario y que se utiliza para automatizar integraciones, acceder a recursos comunes desde fuera de la cuenta o actuar en nombre de un usuario para aprobar elementos en Kissflow. Tiene una función de suplantación que le permite representar a usuarios y realizar acciones en su nombre. Las cuentas de servicio pueden acceder a recursos comunes como Mis elementos, la gestión de usuarios y grupos, procesos, tableros, conjuntos de datos e integraciones, según los casos de uso. Cada acción de la cuenta de servicio se audita en un registro específico, lo que permite a los [Superadministradores y administradores de IAM](/t/35h4ak036h4gg/user-roles-permissions) controlarlas.

![](/migration-assets/service-account-overview/1.png)

> **Nota:**  
> Solo puedes crear un máximo de 25 cuentas de servicio. En cada cuenta de servicio, puedes crear 50 claves de acceso que puedes utilizar para acceder a ellas. 

## Diferencias entre una cuenta de servicio y una cuenta de usuario

Varios [Superadministradores y administradores de IAM](/t/35h4ak036h4gg/user-roles-permissions) pueden administrar las cuentas de servicio, mientras que normalmente son usuarios individuales quienes acceden a las cuentas de usuario. No puedes acceder a las cuentas de servicio mediante el par estándar de nombre de usuario y contraseña, sino autenticando el par de ID y clave secreta de las claves de acceso. Esto reduce el riesgo de acceso no autorizado y las posibilidades de uso indebido. Según el rol del usuario, las cuentas de usuario pueden tener un acceso más amplio a la cuenta de Kissflow, pero las cuentas de servicio solo pueden acceder a los recursos que se hayan compartido con ellas. Por lo general, las cuentas de servicio se crean para un propósito específico, pero una cuenta de usuario no funciona de esa manera. 

## Casos de uso de las cuentas de servicio

Puedes utilizar cuentas de servicio en varios escenarios. A continuación, se muestran algunas formas prácticas de utilizarlas en Kissflow: 

-   Automatizar integraciones entre dos procesos para que el iniciador de un elemento en el primer proceso se considere el iniciador en el segundo proceso y no el propietario de la integración.  
-   Las integraciones autenticadas mediante credenciales de una cuenta de servicio se ejecutarán sin interrupciones, mientras que las integraciones autenticadas mediante credenciales de usuario pueden fallar si el usuario se desactiva o elimina de Kissflow.
-   Acceder mediante programación a recursos internos de Kissflow sin utilizar Kissflow. 

## Ventajas de utilizar una cuenta de servicio

Utilizar una cuenta de servicio ofrece varias ventajas. A continuación, se enumeran algunas de ellas:

-   Las cuentas de servicio pueden automatizar ciertos aspectos de Kissflow sin intervención manual.
-   Puedes conceder acceso selectivo a las cuentas de servicio a recursos específicos dentro de Kissflow, lo que facilita el control y la gestión de los recursos.
-   Las cuentas de servicio mejoran la seguridad al aislar acciones específicas de una cuenta de usuario individual.
-   Las cuentas de servicio ofrecen una forma fiable de acceder a los recursos incluso si el usuario que creó la cuenta de servicio se desactiva o elimina de Kissflow.

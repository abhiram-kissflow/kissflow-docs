✕ Basic ✓ Enterprise

**Event category** is a group of actions, to help admins monitor, review, and audit operations efficiently from the [audit log](/t/h7h4akc/admin-history) page. These categories include process administration, board administration and the like.  
  
An **action** refers to the specific activity carried out by users in an account like dataset permission changes, user activation, or field modifications. Each set of action is mapped to an event category, making it easier to track for events.

This documentation lists all event categories and their corresponding actions.

> **Note:**
> 
> Other actions not listed in this documentation fall under the **unclassified event** category.

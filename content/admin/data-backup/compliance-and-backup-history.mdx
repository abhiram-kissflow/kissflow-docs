# Compliance and backup history

<PlanBadge plans={["enterprise"]} />

<Callout type="info">
  Data backup is available on request. Contact Kissflow support to enable it for your account.
</Callout>

As a [Super Admin](/docs/admin/users/user-roles-and-permissions#super-admin), you manage your account's data backup — you can edit its configuration, cancel a backup that's still in the queue, and review its history.

## Cancel a queued backup

You can cancel a backup only while it's still in the queue, before processing starts. Once a backup is in progress, cancellation is no longer available.

## Edit your backup configuration

While a backup is queued or in progress, its configuration is read-only — you can view it, but not change it. Once the backup completes, the configuration becomes editable again.

1. Sign in to Kissflow.
2. Click your profile picture at the top right of the page, then click **Account administration** > **Data backup**.
3. Click the **More options** button (![More options icon](/migration-assets/compliance-and-backup-history/1.png)) beside **Backup now**, then click **Edit configuration**.

![Data backup page with the More options menu open, showing Edit configuration, Backup history, and Delete configuration](/migration-assets/compliance-and-backup-history/4.png)
{/* TODO: verify screenshot matches current product UI */}

<Callout type="warn">
  Editing your configuration resets your encryption passphrase. Save your previous passphrase somewhere safe before you edit — you'll still need it to open backup files created before the change, and it won't work on any backup you run afterward.
</Callout>

## Track backup progress

While a backup is running, the **Data backup** page shows its real-time status — including stages like validating your configuration, backup in progress, and finalizing the backup — so you always know where an ongoing backup stands.

![Data backup page showing real-time progress through validating configuration, backup in progress, and finalizing data backup](/migration-assets/compliance-and-backup-history/2.png)
{/* TODO: verify screenshot matches current product UI */}

Once a backup completes, its details — including storage mode, backup time, and status — are recorded in the [account-level audit log](/docs/admin/audit-log/audit-log), alongside the user and system actions logged throughout the backup process.

## View backup history

After a backup finishes, you can review its history — the requester, storage mode, data categories included, status, and the date and time of the backup.

1. Sign in to Kissflow.
2. Click your profile picture at the top right of the page, then click **Account administration** > **Data backup**.
3. Click the **More options** button (![More options icon](/migration-assets/compliance-and-backup-history/1.png)) beside **Backup now**, then click **Backup history**.

![Backup history table listing requester, storage mode, data categories, status, and date and time for completed backups](/migration-assets/compliance-and-backup-history/3.png)

# Backing up data to Google Cloud Storage (GCS)

<PlanBadge plans={["enterprise"]} />

<Callout type="info">
  Data backup is available on request. Contact Kissflow support to enable it on your account.
</Callout>

Google Cloud Storage is one of the storage destinations you can configure for account data backups, alongside Amazon S3, Google Drive, and Kissflow's own storage. Once configured, Kissflow exports your account data as JSON files, packages them into ZIP files, and uploads them directly to the GCS bucket you specify.

As a [Super Admin](/docs/admin/users/user-roles-and-permissions#super-admin), you can configure Kissflow to back up your account data to a Google Cloud Storage (GCS) bucket you control.

## Prerequisites

Before you start configuration, grant these permissions in your Google Cloud IAM policy for the target bucket:

- **storage.buckets.get** — returns metadata for the bucket.
- **storage.objects.create** — allows Kissflow to write backup files to the bucket.
- **storage.objects.get** — allows Kissflow to read object metadata from the bucket.

You'll also need a [service account key](https://cloud.google.com/iam/docs/keys-create-delete#iam-service-account-keys-create-console) downloaded as a JSON file — you upload this file inside Kissflow to authenticate.

## Start data backup configuration

1. Sign in to Kissflow.
2. Click your profile picture at the top right of the page, then click **Account administration** > **Data backup**.
3. Click **Start configuration** on the **Data backup** page.

   ![Data backup page with the Start configuration button](/migration-assets/backing-up-data-to-amazon-s3/1.png)
   {/* TODO: verify screenshot matches current product UI */}

Configuration involves three steps: authenticating a storage mode, managing backup settings, and selecting data categories.

## Authenticate your GCS account

1. Select **Google Cloud Storage** as the storage mode from the dropdown.
2. Click **Upload JSON** and select the service account key file you downloaded from Google Cloud. This auto-populates your GCS service account keys.
3. Click **Next**.

   ![Storage mode dropdown with Google Cloud Storage selected and the Upload JSON button](/migration-assets/backing-up-data-to-google-cloud-service-gcs/1.png)
   {/* TODO: verify screenshot matches current product UI */}

<Callout type="info">
  To change your service account keys later, click **Replace JSON** and upload a new key file.
</Callout>

## Manage backup settings

1. **Bucket URI.** Paste the GCS bucket URI where your backup files should be stored.
2. **Select ZIP file size.** Choose **4 GB**, **8 GB**, **16 GB**, or **24 GB**. Files in your account are automatically zipped and split according to your chosen limit.

   For example, a 24 GB limit applied to a 100 GB export produces four 24 GB files and one 4 GB file. Files 24 GB or smaller upload unchanged.

3. **Encrypt backup file** (optional). Switch on this toggle to add a passphrase that encrypts your backed-up files for additional security. Your passphrase must contain:
   - At least eight characters.
   - Both upper and lower case characters.
   - At least one numeric character or symbol.

   <Callout type="warn">
     Save your passphrase somewhere secure. You'll need it to decrypt and access your backed-up files later, and it can't be recovered if lost.
   </Callout>

4. Click **Save**.

   ![Backup preferences with bucket URI, ZIP file size, and encryption options](/migration-assets/backing-up-data-to-google-cloud-service-gcs/4.png)
   {/* TODO: verify screenshot matches current product UI */}

## Select data categories for backup

Choose one or more categories to include:

- **User and group data** — all account user and group data.
- **Transactional data** — flow item details such as status, comments, and assignees.
- **Audit log data** — the last year's audit logs at both the flow and account levels.
- **Attachments** — attachments across flows, including processes, boards, datasets, and apps.

![Data category selection with checkboxes for user and group data, transactional data, audit logs, and attachments](/migration-assets/backing-up-data-to-google-cloud-service-gcs/2.png)
{/* TODO: verify screenshot matches current product UI */}

Select your categories and click **Next**. In the confirmation popup, verify your configuration details and click **Start backup**.

![Backup confirmation popup showing configuration details and the Start backup button](/migration-assets/backing-up-data-to-google-cloud-service-gcs/5.png)
{/* TODO: verify screenshot matches current product UI */}

<Callout type="info">
  Not ready to start? Save your configuration and exit — you can resume it later instead of running the backup immediately.
</Callout>

From the **Data backup** home page, you can also edit your configuration, delete it, or view your backup history.

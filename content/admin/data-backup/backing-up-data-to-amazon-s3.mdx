# Backing up data to Amazon S3

<PlanBadge plans={["enterprise"]} />

Amazon Simple Storage Service (S3) is one of the storage destinations you can configure for Kissflow's data backup feature, alongside Google Drive, Google Cloud Storage, and email delivery. Once configured, Kissflow exports your account data as JSON files, packages them into ZIP files, and uploads them directly to the S3 bucket you specify.

<Callout type="info">
  Data backup is available on request. Contact Kissflow support to enable it for your account.
</Callout>

To configure backup preferences, you need admin access to the **Data backup** section of your account — see [User roles and permissions](/docs/admin/users/user-roles-and-permissions) for how backup access is granted.

## Before you start

Grant the following Amazon S3 bucket permissions in your IAM policy:

- **s3:PutObject** — allows Kissflow to add backup files to your bucket.
- **s3:GetObject** — allows Kissflow to fetch objects from your bucket.
- **s3:ListBucket** — allows Kissflow to list the objects in your bucket.

You'll also need an [AWS access key ID and secret access key](https://docs.aws.amazon.com/IAM/latest/UserGuide/id_credentials_access-keys.html#Using_CreateAccessKey) for authentication. Copy these to your clipboard, or download them as a CSV file — either works when you authenticate inside Kissflow.

## Start a backup configuration

1. Sign in to Kissflow.
2. Click your profile picture at the top right of the page, then click **Account administration** > **Data backup**.
3. Click **Start configuration** on the **Data backup** page.

   ![Data backup page with the Start configuration button](/migration-assets/backing-up-data-to-amazon-s3/1.png)
   {/* TODO: verify screenshot matches current product UI */}

Setting up a backup involves three steps: authenticating your storage destination, managing backup preferences, and selecting the data categories to include.

## Authenticate your Amazon S3 account

1. Select **Amazon S3** as the storage mode from the dropdown.
2. Click **Upload CSV** to auto-populate your **AWS access key ID** and **secret access key**.

   <Callout type="info">
     You can alternatively paste your copied credentials directly into the **Access Key ID** and **Secret Access Key** fields instead of uploading a CSV.
   </Callout>

3. Click **Next**.

   ![Storage mode authentication step with Amazon S3 selected and AWS credential fields](/migration-assets/backing-up-data-to-amazon-s3/2.png)
   {/* TODO: verify screenshot matches current product UI */}

<Callout type="info">
  To change your AWS credentials later, click **Replace CSV** and upload a new CSV file with a different access key ID and secret access key.
</Callout>

## Manage backup preferences

1. **Set the file location.** Paste the S3 bucket URI where you want your backup files stored.
2. **Choose a ZIP file size.** Select **4 GB**, **8 GB**, **16 GB**, or **24 GB**. Kissflow automatically zips and splits your backup according to the limit you choose — for example, a 24 GB limit applied to a 100 GB backup produces four 24 GB files and one 4 GB file. Files 24 GB or smaller upload unchanged.
3. **Encrypt the backup file (optional).** Turn on the switch to add a passphrase that encrypts your backup for additional security. Your passphrase must:
   - Contain at least eight characters.
   - Contain both upper and lower case letters.
   - Contain at least one numeric character or symbol.

   <Callout type="warn">
     Save your passphrase securely. You'll need it to access your backup file later, and Kissflow can't recover it for you.
   </Callout>

4. Click **Next**.

   ![Backup preferences step showing file location, ZIP size, and encryption options](/migration-assets/backing-up-data-to-amazon-s3/3.png)
   {/* TODO: verify screenshot matches current product UI */}

## Select data categories to back up

Choose one or more categories to include in your backup:

- **User and group data** — all account user and group data.
- **Transactional data** — flow item details such as status, comments, and members involved.
- **Audit log data** — the last year of audit logs at both the flow and account levels.
- **Attachments** — attachments in flows such as processes, boards, and more.

![Data category selection step with checkboxes for user and group data, transactional data, audit logs, and attachments](/migration-assets/backing-up-data-to-amazon-s3/4.png)
{/* TODO: verify screenshot matches current product UI */}

## Review and start the backup

1. Select your data categories and click **Next**. A backup confirmation popup opens, summarizing your storage mode, file location, uploaded file size, and selected data categories.
2. Verify the details and click **Start backup**.

   ![Backup confirmation popup summarizing the configuration before starting the backup](/migration-assets/backing-up-data-to-amazon-s3/6.png)
   {/* TODO: verify screenshot matches current product UI */}

<Callout type="warn">
  You can only back up your data once every six months, and a backup can't be canceled once it starts. Review your configuration carefully before clicking **Start backup**.
</Callout>

If you save your configuration and exit without starting the backup, you can resume it later from the same page. From there, you can also edit or delete a configuration, or view your backup history.

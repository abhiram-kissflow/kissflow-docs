✕ Basic ✓ EnterpriseSAML is an acronym for Security Assertion Markup Language (SAML). Its primary function in online security is to allow you to access multiple web applications with a single set of sign in credentials. Setting up SAML-based SSO requires uploading files in Entra ID and Kissflow.

You must be a [Super Admin or an IAM Admin](/t/35h4ak0/user-roles-permissions) to access Account security and set up SAML-based SSO in Kissflow.

To set up SAML-based SSO,

1.  Navigate to **Account Administration** \> **Account settings**. Under **Account security**, click the **Configure SAML** button beside **Sign in with SAML**.![](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/2.png)
2.  The first step is to download the metadata file from Kissflow and upload it to your Entra ID account. Under **Service Provider URLs**, click **Download metadata** to download the metadata from your Kissflow account.![](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/1.png)
3.  Sign in to your  Entra ID account. Then, navigate to your homepage to **create your application**. Provide a name for your application and select **Integrate any other application you don't find in the gallery (Non-gallery)**.
    
    ![](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/8.png)
    
4.  Now, you can create a Single Sign-on method using SAML. Click **Upload metadata file** to upload the metadata file downloaded from Kissflow. Click **Save** once the file has been uploaded.
    
    ![](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/6.png)
    
5.  Scroll down to Federation Metadata XML and click **Download**. The downloaded file should be uploaded inside Kissflow.
    
    ![](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/5.png)
    
6.  Inside Kissflow, click the **Upload XML file** to upload the Federation Metadata XML file downloaded from Entra ID. All the fields under the **IdP configuration** will be populated automatically after the XML file has been uploaded successfully. 
    
    ![](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/3.png)
    
7.  You can make changes under **Advanced settings** to determine the unique identifier for users. **Email address** is the default unique identifier.
    
    > **Note:**  
    > Check the box if you want Kissflow to automatically create new accounts when a new user signs in via SAML-SSO. 
    
8.  Click **Save** to save your SAML configuration in Kissflow. 
9.  Go back to Entra ID to test if your configuration works by clicking **Test this application** on the top navigation bar. If the configuration is successful, Entra ID will lead you to the homepage of your Kissflow account.![](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/7.png)

Once the configuration is completed, toggle the button inside Account Security to enable SAML. You can now add **users and groups** in Entra ID to be authenticated in Kissflow via SAML.![](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/4.png)

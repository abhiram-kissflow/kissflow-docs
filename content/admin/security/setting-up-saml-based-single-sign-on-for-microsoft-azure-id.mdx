# Setting up SAML-based single sign-on for Microsoft Entra ID

<PlanBadge plans={["enterprise"]} />

SAML (Security Assertion Markup Language) lets your users sign in to Kissflow with the same credentials they already use for Microsoft Entra ID, instead of managing a separate Kissflow password. Setting it up is a back-and-forth exchange of metadata files between Kissflow and Entra ID.

<Callout type="info">
  You need to be a [Super Admin or an IAM Admin](/docs/admin/users/user-roles-and-permissions) to access **Account security** and set up SAML-based SSO.
</Callout>

## Download the metadata file from Kissflow

1. Go to **Account Administration** > **Account settings** > **Account security**.
2. Beside **Sign in with SAML**, click **Configure SAML**.

   ![Account security page with the Configure SAML button beside Sign in with SAML](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/2.png)
   {/* TODO: verify screenshot matches current product UI */}

3. Under **Service Provider URLs**, click **Download metadata**.

   ![Service Provider URLs panel with the Download metadata button](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/1.png)
   {/* TODO: verify screenshot matches current product UI */}

This downloads a metadata file that identifies Kissflow as the service provider. You'll upload it to Entra ID in the next step.

## Create the application in Entra ID

1. Sign in to your Microsoft Entra ID account and go to your homepage to create a new application.
2. Name the application, then select **Integrate any other application you don't find in the gallery (Non-gallery)**.

   ![Entra ID dialog for creating a non-gallery application](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/8.png)
   {/* TODO: verify screenshot matches current product UI */}

3. Set up single sign-on using SAML, then click **Upload metadata file** and select the metadata file you downloaded from Kissflow.
4. Click **Save**.

   ![Entra ID SAML setup with the Upload metadata file option](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/6.png)
   {/* TODO: verify screenshot matches current product UI */}

5. Scroll to **Federation Metadata XML** and click **Download**.

   ![Entra ID Federation Metadata XML section with the Download link](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/5.png)
   {/* TODO: verify screenshot matches current product UI */}

You'll upload this file back in Kissflow to complete the exchange.

## Configure the IdP details in Kissflow

1. Back in Kissflow's SAML configuration screen, click **Upload XML file** and select the Federation Metadata XML file you downloaded from Entra ID.

   ![IdP configuration panel with the Upload XML file option](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/3.png)
   {/* TODO: verify screenshot matches current product UI */}

   <Callout type="info">
     Once the upload succeeds, the **IdP configuration** fields — Security key, Sign-out URL, and IdP URL — populate automatically from the XML file.
   </Callout>

2. Under **Advanced settings**, review the **Unique identifier** for users. **Email address** is the default.
3. Check the box to have Kissflow automatically create new accounts when a user who doesn't already exist in Kissflow signs in via SAML-SSO.
4. Click **Save** to save the SAML configuration.

## Test and enable SSO

1. Go back to Entra ID and click **Test this application** on the top navigation bar. If the configuration is correct, Entra ID takes you to your Kissflow account's homepage.

   ![Entra ID application page with the Test this application option](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/7.png)
   {/* TODO: verify screenshot matches current product UI */}

2. Once the test succeeds, go to **Account security** in Kissflow and toggle SAML on.

   ![Account security page with the SAML sign-in toggle enabled](/migration-assets/setting-up-saml-based-single-sign-on-for-microsoft-azure-id/4.png)
   {/* TODO: verify screenshot matches current product UI */}

With SSO enabled, you can add the users and groups in Entra ID that should be authenticated in Kissflow via SAML.

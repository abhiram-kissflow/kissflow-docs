# File upload settings

<PlanBadge plans={["basic", "enterprise"]} />

As a [Super Admin, or an IAM Admin](/docs/admin/users/user-roles-and-permissions), you control which file attachment services account users can upload files from in Kissflow.

## Choose your attachment services

1. Click your profile picture in the upper-right corner of the Home screen.
2. Click **Account administration** > **Account settings** > **File upload settings**.
3. Turn the toggle on or off for each service you want to allow:
   - Google Drive
   - Dropbox
   - Box
   - OneDrive
   - Local files

<Callout type="info">
  At least one service must stay enabled. Kissflow won't let you turn off the last remaining toggle, since users need at least one way to upload attachments.
</Callout>

## How Kissflow secures your attachments

All Kissflow attachments are hosted in Google Cloud Storage (GCS). Data is [encrypted in transit and at rest](https://cloud.google.com/storage/docs/introduction#securing_your_data) using GCS's standard security policies, with Kissflow's application and GCS mutually authenticating each other.

To upload or download an attachment, Kissflow generates a secure link so the file transfer happens directly between the client and GCS instead of routing through additional hops.

Kissflow also blocks manipulated or disguised files from being uploaded. For example, if only JPGs are allowed for a field and someone tries to upload a `.exe` file renamed with a `.jpg` extension, Kissflow detects the mismatch and blocks it. This check is based on the file's metadata — its name, extension, and declared type — rather than inspecting the file's contents, for privacy.

<Callout type="idea">
  If you can't preview an uploaded file, it's likely disguised or invalid. Treat this as a safety flag worth double-checking.
</Callout>

## Validation rules for attachment fields

Beyond account-wide upload settings, admins building a flow's form can add validation rules to individual attachment fields — including [Smart attachment](/docs/build/forms/field-types/image-attachment-smart-attachment-and-signature-fields) fields and attachments on multilingual and public forms. These rules are set on the **Validations** tab of the field:

- **Minimum number of attachments** — the least number of files required for the field.
- **Maximum number of attachments** — the most files a user can add to the field.
- **File type restrictions** — limit uploads to specific formats, including images (JPG, JPEG, PNG, BMP, GIF, TIF), documents (DOC, DOCX, PDF), presentations (PPT, PPTX), spreadsheets (XLS, XLSX), and others (CSV, TXT, EML).
- **Attachment size limit** — cap file size, up to 100 MB.

<Callout type="info">
  For more on configuring these fields, see [Image, Attachment, Smart attachment, and Signature fields](/docs/build/forms/field-types/image-attachment-smart-attachment-and-signature-fields).
</Callout>

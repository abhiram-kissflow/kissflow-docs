# Admin notification settings

<PlanBadge plans={["basic", "enterprise"]} />

**Notification settings** lets Super Admins control how notifications go out across the entire account — which channels are on by default, which server sends the emails, and what those emails look like.

## Global notifications

1. Click your profile picture > **Account administration** > **Notification settings**.

   ![Account administration menu with Notification settings highlighted](/migration-assets/admin-notification-settings/2.png)
   {/* TODO: verify screenshot matches current product UI */}

2. Under **Global notifications**, turn the following toggles on or off for every user on the account:
   - **Email notifications** — sends notifications by email.
   - **Web app notifications** — sends notifications within the web application.
   - **Push notifications** — sends push notifications.
   - **Bot notifications** — sends notifications through the Kissflow bot in Slack, Microsoft Teams, or Google Chat.

<Callout type="info">
  Essential system notifications — such as those related to account security and user management — are prioritized and sent regardless of your channel settings, to make sure nobody misses critical updates.
</Callout>

<Callout type="info">
  The examples below illustrate the kind of system notifications that bypass channel settings. The exact list may vary and isn't limited to these events.
</Callout>

| Notification | Trigger |
| --- | --- |
| User activated | A user's account is activated. |
| User deactivated | A user's account is deactivated. |
| Resend email verification | The verification email is resent. |
| Email address updated | A user's email address is updated. |
| User created | A new user is created. |
| Reset password | A user's password is reset. |
| 2FA status changed | A user's two-factor authentication status is changed. |
| Two factor verification | Two-factor authentication is enabled with an email code. |
| Domain verification | A verification email is sent to connect a domain. |
| Waiting for admin approval | A user creates a process, board, or dataset that requires admin approval. |
| Export | An export related to a user, board, dataform, dataset, or report is carried out. |

## SMTP server settings

<Callout type="info">
  SMTP server settings are available on Enterprise plans only.
</Callout>

By default, Kissflow sends notification emails from its own server. Super Admins can configure a custom SMTP server instead — useful if you want to route emails through your own infrastructure, or if you need to skip email verification for new users.

Click your profile picture > **Account administration** > **Notification settings** > **SMTP server settings**.

![SMTP server settings page showing the Default server and custom server cards](/migration-assets/admin-notification-settings/6.gif)
{/* TODO: verify screenshot matches current product UI */}

### Add a custom SMTP server

1. Click **Add custom SMTP server**.
2. Fill in the server details:
   - **SMTP server name** — a name to identify this configuration.
   - **Host** — your SMTP server's host address.
   - **Port** — the port number.
   - **Security protocol** — **STARTTLS** or **SSL/TLS**.
   - **Authentication type**:
     - **Basic auth** — enter the **Username** and **Password** for your SMTP server.

       ![Basic auth fields for username and password](/migration-assets/admin-notification-settings/1.png)
       {/* TODO: verify screenshot matches current product UI */}
     - **OAuth 2.0** — choose **Google** or **Microsoft** from the provider dropdown. You'll be prompted to sign in and authorize Kissflow to access the email account.

       ![OAuth 2.0 provider dropdown with Google and Microsoft options](/migration-assets/admin-notification-settings/3.png)
       {/* TODO: verify screenshot matches current product UI */}
3. Configure the **Sender profile** — **Sender name**, **Sender email**, and **Reply-to email**.

   By default, this profile applies to every environment (**Development**, **Test**, and **Production**).

   ![Sender profile fields with the environments banner](/migration-assets/admin-notification-settings/4.png)
   {/* TODO: verify screenshot matches current product UI */}

   To override it for a specific environment:

   1. Click the **environments** link in the banner to see your list of environments.
   2. Click **Manage** on the environment you want to customize — for example, **Development**.
   3. Go to **Notification settings** > **SMTP server settings**.
   4. Click **Create profile** — visible once your server is successfully configured — enter the sender details for that environment, and save.

      ![Creating an environment-specific sender profile](/migration-assets/admin-notification-settings/11.gif)
      {/* TODO: verify screenshot matches current product UI */}
4. Under **Server failure alerts**, enter a recipient email address to be notified if the server fails to send emails.
5. Click **Save and test**. Your configuration is saved immediately, and Kissflow runs a connection test in the background — if the test fails, the server card shows a failed validation status, but your saved configuration isn't affected.
6. To switch between the **Default server** and your custom server at any time, click the **More options** icon (![More options icon](/migration-assets/admin-notification-settings/5.png)) on either card and select **Switch to this server**.

   ![Switching between the default and custom SMTP server](/migration-assets/admin-notification-settings/12.gif)
   {/* TODO: verify screenshot matches current product UI */}

### Skip email verification

The **Skip email verification** option lets new users receive notification emails immediately, without first verifying their email address.

<Callout type="warn">
  This is only available with a custom SMTP server — you can't enable it while using the default Kissflow server.
</Callout>

Toggle **Skip email verification** ON on your custom SMTP server card.

![Skip email verification toggle on the custom SMTP server card](/migration-assets/admin-notification-settings/7.png)
{/* TODO: verify screenshot matches current product UI */}

If you haven't configured a custom server yet, switching this toggle prompts you to add one.

![Prompt to add a custom SMTP server when toggling skip verification](/migration-assets/admin-notification-settings/8.png)
{/* TODO: verify screenshot matches current product UI */}

### Edit an active custom SMTP server

You can't change a custom SMTP server's configuration while it's actively sending emails.

1. Click the **Edit** icon (![Edit icon](/migration-assets/admin-notification-settings/9.png)) on the active server card.
2. In the confirmation popup, click **Disable and edit**.
3. Update the server details or sender profile.
4. Click **Save and test**.
5. Once you're ready to use it again, switch back to the custom SMTP server.

<Callout type="info">
  While you're editing, the account temporarily switches to the **Default server** so email delivery isn't interrupted.
</Callout>

### Delete an active custom SMTP server

1. Click the **Delete** icon (![Delete icon](/migration-assets/admin-notification-settings/10.png)) on the active server card.
2. In the confirmation popup, click **Disable and delete**.

<Callout type="info">
  The account automatically reverts to the **Default server** once the custom server is deleted.
</Callout>

### Edit an inactive custom SMTP server

If a custom server isn't currently active, you can update it directly — no disable step required.

1. Click the **Edit** icon on the inactive server card.
2. Update the server details, credentials, or sender profile.
3. Click **Save and test**.

### Delete an inactive custom SMTP server

1. Click the **Delete** icon on the inactive server card.
2. Click **Delete** to confirm.

## Customizing account notifications

Under **Customize notifications**, Super Admins can edit the content of emails sent from the account — the welcome email, the resend-verification email, and the header and footer that wrap every email.

### Editing emails

1. Under the **General** tab, click **Edit** on the welcome email or the resend-verification email.
2. Use the built-in editor to add, remove, or reformat text. Insert custom fields and system fields — like name, email, or phone number — to pull in user-specific information.

   ![Email editor with custom and system fields](/migration-assets/admin-notification-settings/15.png)
   {/* TODO: verify screenshot matches current product UI */}
3. Click **Preview** to check your changes, then **Save**.

<Callout type="info">
  Signature fields aren't supported in these email templates, default or customized.
</Callout>

Click **Reset** at any time to restore the original default content.

### Header and footer

Under the **Header and footer** tab, edit or view the header and footer that appear on every email sent from your account — useful for consistent branding, contact information, or legal disclaimers.

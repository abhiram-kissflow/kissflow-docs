# Geolocation settings

<PlanBadge plans={["basic", "enterprise"]} />

The [Geolocation field](/docs/build/forms/field-types/geolocation-field) lets users capture and view locations on a form using Google Maps. Before anyone can use it, an Account Admin needs to add a Google Maps API key in **Geolocation settings** — without a valid key, the field can't load Google Maps.

![Geolocation settings page with the API key input box](/migration-assets/geolocation-settings/2.png)
{/* TODO: verify screenshot matches current product UI */}

## Access geolocation settings

1. Sign in to Kissflow.
2. Click your profile picture in the upper-right corner of the page.
3. Click **Account administration > Account settings > Geolocation settings**.

## Add or update the Google Maps API key

1. On the **Geolocation settings** page, enter your Google Maps API key in the input box.

   <Callout type="info">
     You can use an existing Google Maps API key or create a new one. Click the **Google Maps API key** link on the page for instructions on Google's developer documentation site.
   </Callout>

2. Save your changes.

<Callout type="warn">
  After you add or update the API key, notify your users to reload Kissflow so the change takes effect.
</Callout>

## When the API key is missing or invalid

If Kissflow can't load Google Maps for the Geolocation field, the message shown depends on where the form is being filled out:

- On the **Kissflow mobile app**, users see a message asking them to contact their Super Admin to add the Google Maps API key.
- On **web**, users see a general connection error instead, without a direct mention of the API key.

![Geolocation field on a form showing a Google Maps loading error](/migration-assets/geolocation-settings/1.png)
{/* TODO: verify screenshot matches current product UI */}

In either case, an Account Admin needs to add or fix the key in **Geolocation settings** to resolve it.

# Cómo incluir IP en la lista de permitidos para conexiones de datos externas

<PlanBadge plans={["enterprise"]} />

Algunas plataformas externas solo aceptan conexiones desde direcciones IP en las que hayas confiado explícitamente. Incluir las IP salientes de Kissflow en la lista de permitidos de esas plataformas les indica que deben tratar a Kissflow como un origen de confianza.

Kissflow envía llamadas salientes —incluidas las llamadas a plataformas externas desde tus integraciones— a través de [Google Cloud NAT](https://cloud.google.com/nat/docs/overview), mediante un conjunto fijo de IP estáticas reservadas.

<Callout type="info">
  No todas las plataformas requieren incluir IP en la lista de permitidos. Consulta primero la documentación de la plataforma externa; si no lo requiere, puedes omitir esta página.
</Callout>

Busca el dominio de tu cuenta a continuación e incluye las IP correspondientes en la lista de permitidos.

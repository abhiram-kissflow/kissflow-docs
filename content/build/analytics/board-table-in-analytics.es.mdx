Las tablas de tablero son tablas generadas por el sistema que aparecen en Kissflow Analytics para cada tablero que crees en tu cuenta de Kissflow. Estas tablas te permiten interpretar y analizar los datos del tablero en Analytics. El módulo Analytics proporciona una vista estructurada y completa de un tablero, incluidos todos los campos de formulario y del sistema. Los subelementos que formen parte de un tablero aparecerán en la tabla principal. Una child table que forme parte de un tablero estará disponible como tabla individual en Analytics.

## Subelementos en una tabla de tablero 

En una tabla de tablero, cada elemento se representa como una fila, y los subelementos también se incluyen como filas dentro de la tabla principal.

Para excluir los subelementos de la vista, puedes aplicar un filtro en la tabla de tablero. También puedes aplicar el mismo filtro en la configuración del informe o en una vista de datos. Por ejemplo, en la página de la tabla de tablero, sigue estos pasos. 

1.  Haz clic en el botón **filter** (![](/migration-assets/board-table-in-analytics/2.jpg)).
    
2.  Selecciona **Add a filter**.
    
3.  Elige el campo **Parent ID** y establece el operador como **Is empty** en el menú desplegable.
    
4.  Haz clic en **Apply** para ver únicamente la lista de elementos principales en la tabla. Como alternativa, si seleccionas el operador **Is not empty**, puedes ver los subelementos de la tabla.
    

![](/migration-assets/board-table-in-analytics/3.png)Las filas se ordenan en la tabla según la hora de modificación, con los elementos más recientes en la parte superior. Esto garantiza que puedas acceder fácilmente a los cambios más recientes realizados en el tablero y hacerles seguimiento.

## Campos del sistema y campos personalizados

En el editor de consultas para crear una vista de datos, los campos del sistema se pueden identificar fácilmente porque utilizan el símbolo '\_' (guion bajo) como prefijo. Algunos ejemplos de campos del sistema son \_id y \_created\_by. Esta convención de nombres también se extiende a las child tables asociadas con el tablero. Los campos personalizados se reconocen porque no comienzan con un guion bajo, aunque pueden tener guiones bajos dentro del nombre del campo. Algunos ejemplos de campos personalizados son Company\_Name y Purchase\_Request\_.

Los campos del sistema se denominan Field IDs en el editor de consultas, mientras que en el visor de fuentes de datos o en la configuración del informe se denominan Field names. Los Field IDs se utilizan para construir consultas, mientras que los Field names se utilizan para ver y configurar informes.

-   **\_id** - Identificador único de cada elemento y subelemento que puedes utilizar como clave principal. 
    
-   **\_item\_id** - El \_item\_id de un subelemento sirve como clave legible que se puede utilizar para filtrar y seleccionar elementos individuales y mostrarlos.
    

Consulta la relación completa entre las entidades de las fuentes de datos para conocer la estructura de las tablas de Analytics.

El acceso a estos campos permite analizar y generar informes para diversos casos de uso. Por ejemplo, puedes calcular el tiempo empleado en los elementos de varios tableros para evaluar el rendimiento del responsable asignado o el estado para medir la eficacia de los empleados.

## Tabla Board Workflow Instance 

La tabla Board Workflow Instance es una tabla del sistema que contiene una descripción general de la información de carga de trabajo, como las actualizaciones de estado de todos los elementos de todos los tableros. Puedes encontrar la tabla del sistema en la pestaña **Data explorer**. También puedes ordenar las fuentes de datos según **System tables** para identificar fácilmente la tabla **Board Workflow Instance**. Puedes utilizar los campos de la tabla Board Workflow Instance para generar varios tipos de informes, como informes tabulares, de gráficos y dinámicos.

> 
> Los cambios de estado de los subelementos no estarán disponibles en la **Board Workflow Instance table**. Esto significa que solo se podrá acceder a los subelementos dentro de su tabla de tablero correspondiente, que proporciona la información más reciente sobre el estado.

## Caso de uso

Considera el siguiente caso de uso para una tabla de tablero. Estás gestionando un tablero que contiene varios elementos en distintas categorías. Puedes crear un informe de distribución por categoría para interpretar los datos y filtrar solo los campos relevantes. Esto te permitirá analizar y visualizar la distribución de los elementos entre las distintas categorías de tu tablero.

![](/migration-assets/board-table-in-analytics/4.png)

Para generar este informe, sigue estos sencillos pasos:

1.  Haz clic en **Create Report**, en la pestaña **Reports** del módulo de Analytics.
2.  Elige **Board Workflow Instance** como fuente de datos, ya que contiene la información necesaria para hacer seguimiento de los estados de los elementos.
3.  Selecciona **Pivot** como tipo de informe para dinamizar y resumir los datos de forma eficaz, y haz clic en **Create**.
4.  Cuando aparezca la página de configuración de la tabla dinámica, establece las columnas como **Category**, ya que serán la base del análisis de distribución.
5.  Elige el campo **ID** como valor, que proporcionará el recuento de elementos de cada categoría.
6.  Establece las filas como **Flow Name** para examinar la distribución según los distintos flujos de procesos.
7.  Para perfeccionar el informe, añade un filtro haciendo clic en **Add a filter**. Elige el campo **Created at**, establece la operación como **Equal to** e introduce el valor **Last 30 days**. Esto limitará el informe a los elementos creados en los últimos 30 días.
8.  **Apply** el filtro y guarda la configuración del informe.
9.  De este modo, obtendrás un informe completo que muestra la distribución por categoría de los elementos creados en los últimos 30 días en el tablero.

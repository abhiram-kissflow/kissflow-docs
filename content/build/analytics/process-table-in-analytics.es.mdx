Las tablas de procesos se generan automáticamente en Kissflow Analytics para cada proceso que crees en tu cuenta de Kissflow. Cada proceso se representa como una tabla que incluye los campos del sistema y los campos personalizados asociados con el proceso. El siguiente diagrama de Analytics, que muestra la relación entre las entidades de las fuentes de datos, puede servirte como referencia.

![](/migration-assets/process-table-in-analytics/1.png)

## Campos del sistema y campos personalizados

Todos los campos de formulario y del sistema asociados con el proceso están disponibles como columnas en la tabla de procesos. El acceso a estos campos permite analizar y generar informes para diversos casos de uso. Los campos del sistema se pueden identificar fácilmente en el editor de consultas para crear [dataview](/t/83h9qzn/understanding-dataviews), ya que usan el símbolo '\_' (guion bajo) como prefijo. Los [campos personalizados](/t/83yqk6f/board-tables-in-analytics#system-fields-and-custom-fields) se reconocen porque no comienzan con un guion bajo, aunque pueden contener guiones bajos en el nombre del campo.

## Child table

Si el proceso tiene child tables, cada [child table](/t/h7h4azm/creating-a-form#child-tablesnbsp) se creará por separado dentro del módulo de Analytics. El sistema genera automáticamente estas tablas en Analytics al crear el proceso en tu cuenta de Kissflow. Estas tablas se generan y nombran automáticamente con el formato «ProcessName-ChildTableName». Por ejemplo, un proceso llamado Purchase Request tendrá una tabla secundaria llamada Purchase Request Catalog Items, donde Catalog items será el nombre de la tabla secundaria creada en el proceso Purchase Request.

## Tablas del sistema

Las [tablas de procesos](https://d56vh6ph4jjmq.cloudfront.net/kissflow/help-articles/static-pdf-process-table-db-mapping.pdf) de Analytics constan de dos tablas del sistema: **ProcessInstance** y **ActivityInstance**. Cada tabla contiene campos específicos del sistema que proporcionan el estado actual y el historial de actividades del flujo de trabajo de todos los elementos de todos los procesos.

Puedes encontrar la tabla del sistema en la pestaña **Explorador de datos**. También puedes filtrar las fuentes de datos según **Tablas del sistema** para identificar rápidamente las tablas. Puedes usar los campos del sistema de las tablas del sistema para generar informes tabulares, de gráficos y dinámicos.

### Tabla ProcessInstance

La tabla de instancias de proceso muestra el estado actual de cada elemento en todos los procesos de tu cuenta de Kissflow. En otras palabras, cada elemento del proceso se muestra en una fila individual de la tabla. Si un proceso tiene ramas paralelas, la columna Parentprocessinstance contendrá el ID del elemento correspondiente para indicarlo.

Este campo se utiliza para determinar los criterios de aprobación dentro de los flujos de trabajo.

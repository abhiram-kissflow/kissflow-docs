## ¿Qué es SAP S/4HANA?

SAP S/4HANA es una base de datos en memoria y una plataforma de desarrollo de aplicaciones que permite procesar grandes volúmenes de datos en tiempo real. Combina capacidades transaccionales y analíticas en un solo sistema para agilizar la toma de decisiones y simplificar los entornos de TI de empresas de todos los tamaños.

## Acerca de la integración de SAP S/4HANA con Kissflow

El conector de SAP S/4HANA en Kissflow Integration permite interactuar sin problemas con la base de datos en memoria de SAP S/4HANA, lo que posibilita realizar operaciones de datos en tiempo real directamente desde las aplicaciones y los flujos de trabajo de Kissflow. Para integrarse con SAP S/4HANA, debes tener acceso a una instancia válida de SAP S/4HANA mediante API y ser cliente empresarial de pago de Kissflow.

Por ejemplo, un equipo de compras crea una solicitud de compra de proveedor en un proceso de Kissflow para adquirir materias primas. Una vez que la solicitud se aprueba internamente, Kissflow activa una llamada API a SAP S/4HANA para insertar un nuevo registro de solicitud de compra. La carga útil incluye los detalles del proveedor, las especificaciones de los artículos, las cantidades, los precios y las fechas de entrega requeridas. Esta sincronización en tiempo real garantiza que SAP S/4HANA refleje de inmediato las solicitudes de compra aprobadas, lo que permite realizar un seguimiento, elaborar presupuestos y generar informes precisos en todos los departamentos.

Actualmente, no hay triggers disponibles para el conector de SAP S/4HANA.

## Acciones en el conector de SAP S/4HANA

**Acción** 

**Descripción**

Crear una solicitud de compra

Esta acción inicia una nueva solicitud de compra.

Realizar una solicitud API 

Esta acción realiza una solicitud API a tu cuenta de SAP S/4HANA.

Obtener la lista de proveedores

Esta acción recupera la lista completa de proveedores disponible.

Obtener los datos maestros de materiales

Esta acción recupera la información general asociada con los materiales del registro maestro.

### Configurar el conector de SAP S/4HANA

1.  En tu cuenta de Kissflow, haz clic en el botón **Crear** del panel de navegación izquierdo y selecciona Integration en la lista. Introduce un nombre y una descripción únicos para tu integración y haz clic en [Crear](/t/60h4akx/creating-an-integration). 
    
2.  Una vez dentro del editor, configura un paso de trigger, como un evento de trigger de Kissflow Process, Cuando un elemento completa su flujo de trabajo.
    
3.  Después de configurar el paso de trigger anterior, haz clic en el **botón Agregar** (![](/migration-assets/sap-s4hana-connector/2.png)) para configurar el paso de acción. Busca **SAP S/4HANA** y selecciona el evento de acción **Crear una solicitud de compra**.
    
4.  Después de elegir el evento de acción, autentica el conector. Haz clic en **\+ Agregar una cuenta**. Introduce el nombre de usuario y la contraseña de tu cuenta de SAP S/4HANA para autenticarte.
    
5.  Después de autenticarte con tu cuenta de SAP S/4HANA, agrega los valores relevantes en los campos del formulario de SAP S/4HANA. Puedes hacer clic en **Actualizar campos** para ver todos los campos actualizados del formulario seleccionado. También puedes asignar campos del paso de trigger o campos de los pasos de acción anteriores. Estos valores se agregarán automáticamente a los campos del elemento después de ejecutar la acción.
    
6.  Después de una configuración correcta, comprueba si el paso de acción de SAP S/4HANA está configurado correctamente haciendo clic en el botón Probar. El conector extraerá los datos de los campos de la cuenta de SAP S/4HANA seleccionada y los mostrará como una salida JSON. Al igual que con el paso de trigger, recibirás una notificación inmediata de los errores. Corrígelos antes de volver a probar el paso de acción.
    

![](/migration-assets/sap-s4hana-connector/1.png)

### Configurar un conector de SAP S/4HANA

Después de crear un flujo de trabajo personalizado con los pasos de acción del conector de SAP S/4HANA, activa el botón de alternancia para activar la ejecución de la integración. Cuando se produce un evento de trigger, se ejecuta la acción de SAP S/4HANA.

Para asegurarte de que tus integraciones estén configuradas y probadas correctamente, consulta nuestra [guía de prácticas recomendadas](/t/p8yzsxn/best-practices-for-using-kissflow-integrations) para obtener consejos sobre cómo crear soluciones sólidas y fáciles de mantener.

# Filtering items in a process

<PlanBadge plans={["basic", "enterprise"]} />

Instead of scrolling through every record, apply a filter to your process page to see only the items that match specific conditions. You can filter items inside the **Draft**, **In progress**, **Completed**, **Withdrawn**, and **Rejected** tabs.

## Add a filter

A filter can be a single condition, or a set of conditions grouped together. Only items that satisfy the filter are shown on the process page.

1. Click the **Filter** button on your process page.

   ![Filter button on the process page toolbar](/migration-assets/managing-filters/5.png)
   {/* TODO: verify screenshot matches current product UI */}

2. Click **+ Add a filter**.
3. Select a field from the **Select field** dropdown, then define the condition:
   - Choose an operator. Operators are contextually populated based on the field you select — for example **Equal to**, **Not equal to**, or **Contains** for a text field, and **Is empty** where applicable.
   - Select a field or enter the value to check the condition against.

   ![Operator dropdown populated based on the selected field's type](/migration-assets/filtering-items-in-a-process/6.gif)
   {/* TODO: verify screenshot matches current product UI */}

4. To add more conditions to the same group, click **Add new condition** and choose whether the group uses **AND** or **OR** logic.

   <Callout type="info">
     A filter group can use only one logical operator throughout. If a group has five conditions, they must all be joined with AND, or all with OR — you can't mix AND and OR within the same group.
   </Callout>

5. Click **+ Add new filter** to add as many filter groups as you need. Choose the logical operator that applies across the groups: **AND** requires every group to be true, and **OR** requires at least one group to be true.

   ![Multiple filter groups joined by an AND/OR operator](/migration-assets/filtering-items-in-a-process/1.png)
   {/* TODO: verify screenshot matches current product UI */}

6. Click **Apply** to filter the items.

## Filter by the current user

When you're matching a text or user field against another field, use the **Current User** option to reference the person viewing the filter, instead of a fixed name.

While setting the value for a condition, choose **Current User** in the right-hand dropdown, then pick the attribute that fits your condition: **Name**, **Email address**, **Manager**, **Status**, or **Job Title**.

![Current User option with its attributes in the filter value dropdown](/migration-assets/filtering-items-in-a-process/3.png)
{/* TODO: verify screenshot matches current product UI */}

For example, to check whether a department head is the person viewing the filter, set the condition to:

**Department head** — **Equal to** — **Current User.Name**

## Filter by relative dates

For date and time fields, you can filter against a relative value instead of a fixed date. While matching a date or time field against a value, click the **More options** button to see the relative options:

## Generate filters with AI

<PlanBadge plans={["enterprise"]} />

<iframe src="https://player.vimeo.com/video/1133474101?app_id=122963" title="AI-powered advanced filters - Overview video" className="aspect-video w-full rounded-lg border" allowFullScreen />

Instead of manually building conditions, describe what you want to see in plain language and let AI build the filter for you.

1. Click the **Filter** button on your process page.

   ![Filter button that opens the filter pane](/migration-assets/adding-advanced-filters-using-ai/1.png)
   {/* TODO: verify screenshot matches current product UI */}

2. In the filter pane, click **Advanced filter**.
3. In the text box under **Generate advanced filter using AI**, type your filtering criteria in plain language — for example, _My high priority assigned items_.
4. Click **Generate**. AI creates the corresponding filter conditions in the section below.
5. Once you're satisfied with the conditions, click **Apply** to see your filtered results.

<Callout type="info">
  AI-powered filter generation is available on the Enterprise plan. On the Basic plan, you can still build filters manually as described above.
</Callout>

### Summarize and regenerate filters

Click **Filter summary** to get an AI-generated summary describing what your filter conditions represent. You can use this summary as a base for new prompts:

1. Click **Edit summary**.
2. Update the summary text to reflect your new requirement.
3. Click **Regenerate** to make AI create a new set of filter conditions.

For example, if the summary says _This filter shows items with priority High that are assigned to the current user_, you can edit it to say _This filter shows items with priority Critical that are assigned to the current user_ and regenerate the filter.

### Example prompts

| To see... | You can type... |
| --- | --- |
| High-priority items assigned to you | "My high priority assigned items" |
| Items that are past their due date | "Show me all overdue tasks" |
| Items created within the current week | "Everything created this week" |
| Items assigned to a specific person | "Tasks assigned to Natasha Romanoff" |
| Items with multiple conditions | "All critical items in the approval step due today" |

### Troubleshooting

**Error: "Filter generation failed: Some required fields are hidden or couldn't be selected. Please make them visible in the Show/hide fields menu or refine your prompt."**

- **Why it happens** — AI can only build filters using fields that are currently visible on your process. If your prompt mentions a field like "Name" or "Asset type" but that field is hidden in **Show/hide fields**, AI can't access it and the filter fails.
- **How to fix it**:
  1. Close the filter pane.
  2. Click the **Show/hide fields** button.

     ![Show/hide fields button](/migration-assets/filtering-items-in-a-process/4.png)
     {/* TODO: verify screenshot matches current product UI */}

  3. In the dropdown, select the checkbox for every field you want to filter by.
  4. Try generating the AI filter again.

**The generated filters aren't what I expected.**

- **Why it happens** — AI may have misinterpreted an unclear request.
- **How to fix it** — rephrase your request to be more direct and specific. For example, instead of _Show old tasks_, try _Show tasks created before October 1, 2025_. Including field names in your request often leads to more accurate results.

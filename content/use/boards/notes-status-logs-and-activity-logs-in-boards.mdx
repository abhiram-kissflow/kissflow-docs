# Notes, status logs, and activity logs in boards

## Notes

Notes offer a way to collaborate with stakeholders on an item. Add comments, log updates or action items, and react to notes added by others — visible to anyone collaborating on the item.

To add or update a note, open an item and click the **Notes** tab in the right navigation strip. Type **/** in the text area to open the formatting menu — attach a link, add a bulleted or numbered list, or apply Bold, Italic, Underline, Strikethrough, Code snippet, and Blockquote formatting.

You can also add emojis, attach files, images, and audio (played directly in Kissflow without downloading), annotate uploaded images, and tag people with **@mentions** — the tagged user gets a notification. Click the thumbs-up icon to acknowledge someone else's note, or copy a note's link to share it with others.

<Callout type="info">
  Once an item has at least 10 notes, a **Summarize** button appears at the top of the **Notes** tab to generate a quick AI summary of the recent notes.
</Callout>

## Status log

The status log captures every status change made to an item, along with the time spent in each status. Open an item, click the **Item log** tab in the right navigation strip, and select the **Status log** subtab.

Use the time spent in each status to spot items that are stuck — especially critical ones — so they get resolved without delays.

## Activity log

The **Activity log** subtab of the **Item log** tab lists key actions performed on an item — changes to system fields such as **Priority**, **Assignee**, **Start date**, and **Due date** (with the value before and after each change), plus when the item was created, closed, or cloned. Changes to custom fields are not shown here.

Each entry includes:

- **Timestamp** — the exact time the action was performed.
- **Performed by** — the member who performed the action.
- **Activity** — the type of action, with details of any changes it caused.

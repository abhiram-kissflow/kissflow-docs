In mid-January 2024, we will deprecate the employee management table in Kissflow. Post this, you will no longer be able to access employee details through this table.  
 

**Transition to User Management Table:**  
All relevant details, previously available in the employee management table, will now be accessible through the user management table. Please use the user management table to view and manage employee information.  
 

**Changes in Data Sharing:**  
While the employee management table allows users to share employee details with others, the user management table will have different access restrictions. Only User admins / IAM admins / Super admins will have the ability to access and view user details. This change is implemented to enhance data security and privacy.  
  
**Action Required:**  
Please familiarize yourself with the user management table. If you have been sharing employee details with other users, kindly inform them of this change and advise them to contact User admins / IAM admins / Super admins for any required information. Also, note that you will not be able to use the \_employee() system field in the future. Existing fields with \_employee() will be replaced with user() automatically.  Moving forward, you can use user() in your new workflows.

We understand that changes like these can impact your daily operations, and we appreciate your cooperation during this transition period. If you have any questions or concerns, please feel free to reach out to our support team in the community. 

**Note:** This applies to users in the following plans - KF Digital Workflow, KF Process, KF Project, KF Workflow Lite, KF Workflow Suite, and KF Workflow.

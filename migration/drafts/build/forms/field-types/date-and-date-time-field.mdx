The format of both the **Date** and **Date & Time** fields is based on the locale set up by the user. If nothing is set, then the account locale settings will be used.

## Date field

The Date field allows you to pick a date from the date picker.

### Settings

You can set a default date for the field to display when the form first loads.

### Validations

There are nine types of validations you can use for a date field. [Learn more about all the validations](/t/m1h4az4/field-validations).

### Expressions/formulas

There are many [expressions and formulas you can use with a date field](/t/p8h4al3/date-expressions).

## Date & Time field

The Date & Time field works exactly like a date field and lets you collect the time.

When hovering over the field, the time zone will display. It will show the user's set time zone, or if the user has not set a time zone, the account time zone.

### Settings

You can set a default date and time for the field to display when the form first loads.

### Validations

You can use nine types of validations for a date and time field. Learn more about all the [validations](/t/m1h4az4/field-validations).

### Expressions/formulas

There are many [expressions and formulas you can use with a date & time field](/t/p8h4al3/date-expressions).

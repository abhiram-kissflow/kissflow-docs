## What is a custom trigger?

In cases where you can't set up an integration using an existing trigger, you can configure and set up a custom trigger for a specific workflow. A custom trigger allows you to define what input the trigger will look for to start other integration actions by defining your own input parameters. There are several instances where a custom trigger might be beneficial over a predefined Kissflow trigger. 

-   Currently, you cannot call an integration inside another integration. With a custom trigger, you can call your custom trigger as an action in a different integration, triggering the execution of actions added under the custom trigger. 
    
-   A custom trigger is not restricted to the action. It can be triggered or invoked from different flows with access to this custom trigger. 
    

Assume an example where there are two processes, **Employee onboarding,** and **Employee offboarding**, and there is an **Employee master** dataset. When an employee enters or leaves the company, the changes must be updated in the Employee master dataset. Previously, it was only possible to use two different integrations. Using a custom trigger, you can do it with a single integration. 

## Custom trigger as a trigger step

**Trigger**

**Description**

Set up a custom trigger

A customized trigger that can be configured with input fields and tables.

1.  In your Kissflow account, click the **Create** button in the left navigation panel and select **Integration** from the list. Enter a unique name and description for your integration and click **[Create](/t/60h4akx/creating-an-integration)** next to the Custom trigger section. 
    
2.  Choose the flows and integrations that can access this custom trigger. Flows that do not have access to this custom trigger won’t be able to execute actions under it. 
    
3.  Create input fields or tables, name them, and you must choose a data type from the four available types.
    
4.  Once you have created the input fields and configured the input parameters, click **Next** and **Done** to save the configuration. The trigger step of the custom trigger does not require authentication using your Kissflow account. You can add actions after this trigger step that needs to be executed.
    

Once you have created the custom trigger, it can be called an action in different integrations. When the integration with the custom trigger as an action runs, the run history will contain all the flow-specific details.

## Custom trigger as an action step

**Action**

**Description**

Call an integration

This action triggers the custom trigger and executes the action(s) added under it.

1.  Click the Custom trigger by Kissflow connector and select the action step: **Call an integration or flow.** 
2.  Select the custom trigger that you created in the trigger step. The trigger part of the custom trigger can be added to more than one integration. 
    
3.  Map the pre-configured input fields using the data from above actions to trigger the actions below. 
    
4.  Check whether the action step is set up correctly. Click **Use sample data** to use sample data to validate the action. The connector will validate sample data against the fields in the board and display results in JSON format. Errors, if any, are notified to you right away. Fix them before retesting the action.
    

![](/migration-assets/custom-trigger-by-kissflow/2.png)

After setting up the required triggers and actions, turn **ON** the integration to activate it.

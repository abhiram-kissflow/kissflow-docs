Kissflow automatically captures data about every item that goes through a flow. For example, when a user creates a new item, Kissflow automatically stores who created the item and what time it was created. 

This data is stored in system fields. The values in system fields cannot be manually set or edited, they are only system-generated. However, they can be displayed or used in expressions and reports. 

Each type of flow has a specific set of system fields associated with it. If you need any help using these fields, please reach out to our support team.

#### Notes

1 Processes have five possible statuses:

-   Draft
-   In progress
-   Completed
-   Rejected
-   Withdrawn

Projects and cases have four:

-   Not started: The item has not been moved to another step yet
-   In progress: The item is active on the board
-   Archived. The item is not active on the board
-   Completed: The item has been moved to the completed step

2 There are three possible states for projects:

-   In progress
-   On hold
-   Done

3 [User fields require additional methods](https://helpdocs.kissflow.com/expression/user-expressions) to display in forms.

## How to display system fields on your form

1.  Add a new field of the correct type to the form. 
2.  Turn on **Is this a computed field?**
3.  Enter the name of the system field as described above.

## Viewing system fields in reports

All of the system fields are available to use for your reports. When making a report, there is a special section of fields called **System** that you can choose from. 

![](/migration-assets/system-fields/1.png)

## Using system fields in expressions

You can use system fields in expressions and the dot operator to call up additional information depending on the data type. [Learn more about expressions here](https://helpdocs.kissflow.com/expression/introduction-to-expressions-and-formulas).
